JAVA 기초

용어 정리

코덱 = 인코딩 + 디코딩
인코딩 : 사람 말 → 컴퓨터 말
디코딩 : 컴퓨터 말 → 사람말
저급 ~ 고급 언어 존재. C는 중간 점. 대부분 알고 있는 JS, python, JAVA는 고급언어임.
기준 : 인간이 알아들을 수 있느냐?

자바 특징

1. 이식성이 매우 높은 언어 : 소스코드파일 → 한번의 컴파일 → 클래스파일 → 윈도우, 리눅스, 맥 가리지 않고 다 가능
2. 객체 지향 언어(OOP, Object Oriented Programming) : 작은 파일부터 만들고, 조합해 전체 프로그램을 완성하는 기법
- 객체? 사물 개념. ex) 모니터 바라보는 속성이 있어야함. 인치, 버튼, 해상도 등.
기능 - 디스플레이, 사운드, 티비 기능 등 → 클래스화 해서 쓰는 것.
3. 캡슐화, 상속, 다형성 기능을 완벽하게 지원 : 상속 → 재활용. 부품들을 모아서 만든다?
4. 메모리 자동 관리 기능이 강력해짐
- 사용하지 않는 객체를 자동으로 메모리에서 제거 : GC(garbage collecter) - 핵심 기능 코드에 집중할 수 있도록 하는 기능
- C,C++은 메모리를 개발자가 직접 조작을 해야 함
5. 함수적 스타일 코딩 지원 - 람다식 지원.
- 람다식으로 인하여 코드 간결하게 작성 가능 (10줄 쓸걸 1줄로)
- 컬렉션(객체의 집합) 요소를 필터링, 매핑, 그룹핑, 집계 처리 등 저용량 코드로 고성능을 냄
6. 웹, 앱 등 다양한 애플리케이션 프로그램의 개발 가능
7. 개발 진행중, JDK(JAVA development kit) 1.12까지 발표
8. 멀티스레드 기능 구현이 쉬워짐 (스레드, 작업단위), 대용량 작업, 동시 여러기능 가능.
9. 동적 로딩(Dynamic Loading) 지원 : 미리 객체를 만들어 놓지 않고 필요한 시점에 동적 로딩, 유지 보수시 특정 객체만 쉽게 수정 및 교체 가능
10. 풍부하고 엄청난 오픈소스 라이브러리를 제공

자바 프로그램 개발의 순서

1. 프로그램 시작점 : 메인 - 오로지 하나. _____() : 메소드, 함수
2. 주석 : //, 어차피 기억못하니까 줄마다 주석쳐서 설명달아놓을 것 /* ~ */ 범위 주석
3. 실행문과 세미콜론(;) : 변수 선언, 값저장, 메소드 호출에 해당하는 코드. 실행문 끝에는 반드시 ; 을 붙여 실행문의 끝 표시

eclipse

- 자바 통합개발환경(IDE), IBM에서 개발.
- 지속적으로 버전업, 배포, 플러그인 프로그램 설치 가능
- 안드로이드, 스프링, JSP, C++ 등 통합개발환경 구축 가능
- 자바로 개발된 도구임 - 워크스페이스 : 프로젝트가 기본적으로 저장되는 폴더. 상황별로 다르게 지정 가능.
- metadata 폴더 : 자동적으로 생성. 메타데이터 저장, 삭제시 이클립스 재실행하면 초기상태로 됨.
(메타데이터 : 폰트 설정 등등) - 아파트 동이 자바 작업 폴더, practice폴더는 몇호인지, 안에 방은 패키지 파일, 방안의 가구는 자바 파일들.
